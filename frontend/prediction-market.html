为什么要精简

技术侧的处理逻辑

产品提供的需求表（见飞书表格截图）中，重复逻辑较多——同一个 btn_name 在多个页面重复列出（如 switch_market_category 出现 2 次、market_detail 出现 3 次），以及拆分过细的操作事件（关闭面板、输入金额等）。

全量埋点的代价：高频无意义事件（如输入金额每次按键都触发）会产生大量噪音数据，淹没真实的转化信号，增加 BI 清洗成本；重复事件造成指标混乱，难以做归因分析。

优化原则：① 合并——相同语义事件用参数字段区分（如体育筛选合并为一个事件，通过 filter_type 区分大类 / 联赛）；② 删除——被动关闭行为 / 纯 UI 偏好操作 / 可由其他事件反推的行为；③ 复用——金额等信息随 Callback 事件一并上报。

删除的 4 个事件

以下事件从产品需求表原版（33 个）中删除，最终保留 29 个。

删除 btn_name原始描述删除原因
close_trade_panel 关闭右侧交易面板 被动关闭行为，可通过 buy_position 漏斗缺口反推，无独立分析价值
close_sell_position_modal 关闭卖出弹窗（无按钮标识） 与 cancel_sell_position 完全重复，合并后由 close_source 字段区分来源
input_trade_amount 输入交易金额 高频触发（每次按键），产生大量噪音；金额信息随 buy_position 一同上报即可
toggle_asset_visibility 显示/隐藏账户资产 纯 UI 偏好操作，不影响业务漏斗或转化分析
⚠ 待对齐事项：以上删减需要与 BI 侧确认，确保不影响现有指标体系。如 BI 有特定漏斗分析需求，可在本文档补充说明后恢复对应事件。
